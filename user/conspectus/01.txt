
---------- Инциализыция проекта NPM ----------
-- инициализируем npm проект
в корневом каталоге проекта 
npm init

Перед тем как установим процессор в проект, отключим расширение в VSCode которое компилирует
.scss в .css он же watcher
расширение называется Live Sass Compiler 
Отключим расширение потому, что мы будем делать компиляцию и whatching по-другому. 

-- устанавливаем препроцессор sass версии 1.80.6
npm i sass@1.80.6 -D

чтобы комфортно работать с препроцессором стилей нужно добавить специальную команду 
"sass-watch": "sass --watch ."

в файле package.json в разделе "scripts"

Этой командой мы запустим препроцессор sass для того, чтобы он автоматически отслеживал 
изменения во всех файлах .scss в рамках папки текущего проекта, и компилировал их в файлы .css

Препроцессор запускается один раз, а дальше он работает в автоматическом режиме. 
запускаем команду 
npm run sass-watch

теперь когда команда запущена, каждый раз при изменениях в файлах .scss будет 
происходить компиляция в файлы .css 
появилось 2 файла:
main.css и main.css.map 

подключаем файл main.css в index.html 

нижнее подчеркивание в файлах .scss говрят о том, что эти файлы обрабатывать не нужно,
то есть для этих файлов не будут дублироваться файлы .css
